# 43 — OpenCode: следующие задачи (без ZCode, координация через Claude)

## Контекст
ZCode больше не участвует в проекте. Координация — напрямую через Claude
(в чате) + ты выполняешь на диске и присылаешь результат/логи.

## Задача 1 — Paywall: обратная связь при недоступном сторе (релиз)
В `purchase_provider.dart` метод `purchasePlan()`:
```dart
if (!available) {
  if (kDebugMode) {
    await _simulatePurchase(planType);
  }
  return;   // ← в релизе просто молча ничего не происходит
}
```
и аналогично при `product.id != productId`.

**Нужно:** в `paywall_screen.dart` добавить обработку пустого результата —
если `purchasePlan()` завершился без активации Premium и без ошибки,
показать `SnackBar`/диалог: "Магазин недоступен, попробуйте позже" (текст
через `AppStrings`/`AppStringsLocale`, RU+EN).

Технически: `purchasePlan()` должен возвращать `bool` (успех/неуспех
инициации покупки, не факт покупки — покупка асинхронна через stream),
чтобы `paywall_screen.dart` мог показать сообщение при `false`.

## Задача 4 — cycle_data.dart: локализация строк
`phaseOn()` возвращает захардкоженные русские строки
(`'Менструация'`, `'Фертильное окно'`, `'Фолликулярная фаза'`,
`'Лютеиновая фаза'`) напрямую, не через `AppStrings`. На английской
локали это будет выглядеть неправильно. Вынести в `strings.dart`/
`strings_en.dart`, передавать нужный `AppStringsLocale` в метод.

## Задача 5 — year_overview_screen.dart: цвета не из темы
```dart
final color = switch (avg) {
  > 50 => Colors.green.shade300,
  > 0 => Colors.green.shade100,
  > -50 => Colors.red.shade100,
  _ => Colors.red.shade300,
};
```
Хардкод `Colors.green`/`Colors.red` вместо палитры из `AppColors`/темы —
низкий контраст в тёмной теме. Не блокер, но стоит поправить после
приоритетных задач 1-3.
